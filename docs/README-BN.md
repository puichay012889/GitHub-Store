# 🗺️ প্রকল্প সংক্ষিপ্ত বিবরণ

GitHub Store হলো GitHub রিলিজের জন্য একটি ক্রস-প্ল্যাটফর্ম অ্যাপ স্টোর, যা ওপেন-সোর্স সফটওয়্যার আবিষ্কার ও ইনস্টলেশনকে সহজ করার জন্য ডিজাইন করা হয়েছে। এটি স্বয়ংক্রিয়ভাবে ইনস্টলযোগ্য বাইনারি (APK, EXE, DMG, AppImage, DEB, RPM) শনাক্ত করে, এক ক্লিকে ইনস্টলেশন প্রদান করে, আপডেট ট্র্যাক করে এবং রিপোজিটরি তথ্য একটি পরিচ্ছন্ন অ্যাপ স্টোর-স্টাইল ইন্টারফেসে উপস্থাপন করে।

Kotlin Multiplatform এবং Compose Multiplatform ব্যবহার করে Android ও Desktop প্ল্যাটফর্মের জন্য নির্মিত।

## 🚀 বৈশিষ্ট্যসমূহ

- **স্মার্ট অনুসন্ধান**
    - "Trending", "Hot Release" এবং "Most Popular" প্রকল্পগুলোর জন্য হোম বিভাগসমূহ, সময়-ভিত্তিক ফিল্টারসহ।
    - শুধুমাত্র বৈধ ইনস্টলযোগ্য ফাইলসহ রিপোজিটরিগুলো প্রদর্শিত হয়।
    - প্ল্যাটফর্ম-সচেতন টপিক স্কোরিং যাতে Android/Desktop ব্যবহারকারীরা প্রাসঙ্গিক অ্যাপগুলো প্রথমে দেখেন।
    - উন্নত প্রাসঙ্গিকতা র‍্যাঙ্কিং ও পারফরম্যান্সসহ নতুনভাবে ডিজাইন করা সার্চ।

- **রিলিজ ব্রাউজার ও ইনস্টলেশন**
    - শুধু সর্বশেষ নয়, যেকোনো রিলিজ থেকে ব্রাউজ ও ইনস্টল করার জন্য রিলিজ সিলেক্টর।
    - প্রতিটি রিপোজিটরির সকল রিলিজ আনে।
    - একক "সর্বশেষ সংস্করণ ইনস্টল করুন" অ্যাকশন, সাথে সকল উপলব্ধ রিলিজ ও তাদের ইনস্টলারের ড্রপডাউন তালিকা।
    - স্বয়ংক্রিয় সামঞ্জস্যতা পরীক্ষাসহ ম্যানুয়াল ইনস্টলেশন বিকল্প।

- **বিস্তারিত তথ্য পৃষ্ঠা**
    - অ্যাপের নাম, সংস্করণ, "সর্বশেষ সংস্করণ ইনস্টল করুন" বোতাম এবং শেয়ার অ্যাকশন।
    - স্টার, ফর্ক, খোলা ইস্যু।
    - রেন্ডার করা README বিষয়বস্তু ("এই অ্যাপ সম্পর্কে")।
    - নির্বাচিত যেকোনো রিলিজের জন্য Markdown ফরম্যাটে রিলিজ নোট।
    - প্ল্যাটফর্ম লেবেল ও ফাইল সাইজসহ ইনস্টলারের তালিকা।
    - ডিপ লিংক সাপোর্ট — URL-এর মাধ্যমে সরাসরি রিপোজিটরি বিবরণ খুলুন।
    - একজন ডেভেলপারের রিপোজিটরি ও কার্যকলাপ দেখার জন্য ডেভেলপার প্রোফাইল স্ক্রিন।

- **অ্যাপ ব্যবস্থাপনা**
    - GitHub Store থেকে সরাসরি ইনস্টল করা অ্যাপ খুলুন, আনইনস্টল করুন এবং ডাউনগ্রেড করুন।
    - Android: APK আর্কিটেকচার ম্যাচিং (armv7/armv8), প্যাকেজ মনিটরিং এবং আপডেট ট্র্যাকিং।
    - Desktop (Windows/macOS/Linux): ব্যবহারকারীর ডাউনলোড ফোল্ডারে ইনস্টলার ডাউনলোড করে এবং ডিফল্ট হ্যান্ডলার দিয়ে খোলে।

- **তারকাচিহ্নিত রিপোজিটরি**
    - অ্যাপ থেকে আপনার তারকাচিহ্নিত GitHub রিপোজিটরিগুলো সংরক্ষণ ও ব্রাউজ করুন।

- **নেটওয়ার্ক ও কর্মক্ষমতা**
    - কনফিগারযোগ্য নেটওয়ার্ক রাউটিংয়ের জন্য ডায়নামিক প্রক্সি সাপোর্ট।
    - দ্রুত লোডিং ও কম API ব্যবহারের জন্য উন্নত ক্যাশিং সিস্টেম।

- **ক্রস-প্ল্যাটফর্ম UX**
    - Android: নেটিভ স্প্ল্যাশ স্ক্রিন, সেশন মেয়াদ শেষ হ্যান্ডলিং এবং অ্যাডাপ্টিভ আইকন।
    - Desktop: DEB ও RPM ফরম্যাটের পাশাপাশি Linux-এ AppImage সাপোর্ট অগ্রাধিকার।
    - ১২টি ভাষায় অনূদিত: ইংরেজি, স্প্যানিশ, ফরাসি, জাপানি, কোরিয়ান, পোলিশ, রাশিয়ান, চীনা, বাংলা, হিন্দি, ইতালীয় এবং তুর্কি।

## 🔍 আমার অ্যাপ কিভাবে GitHub Store-এ প্রদর্শিত হয়?

GitHub Store কোনো ব্যক্তিগত ইনডেক্সিং বা ম্যানুয়াল কিউরেশন নিয়ম ব্যবহার করে না।
আপনার প্রকল্প এই শর্তগুলো পূরণ করলে স্বয়ংক্রিয়ভাবে প্রদর্শিত হতে পারে:

1. **GitHub-এ পাবলিক রিপোজিটরি**
    - দৃশ্যমানতা অবশ্যই `public` হতে হবে।

2. **সর্বশেষ রিলিজে ইনস্টলযোগ্য ফাইল**
    - সর্বশেষ রিলিজে কমপক্ষে একটি সামঞ্জস্যপূর্ণ এক্সটেনশনের ফাইল থাকতে হবে:
        - Android: `.apk`
        - Windows: `.exe`, `.msi`
        - macOS: `.dmg`, `.pkg`
        - Linux: `.deb`, `.rpm`, `.AppImage`
    - GitHub Store স্বয়ংক্রিয়ভাবে তৈরি সোর্স কোড আর্টিফ্যাক্ট (`Source code (zip)` /
      `Source code (tar.gz)`) উপেক্ষা করে।

3. **সার্চ / টপিকের মাধ্যমে আবিষ্কারযোগ্য**
    - রিপোজিটরিগুলো GitHub-এর পাবলিক সার্চ API-এর মাধ্যমে আনা হয়।
    - টপিক, ভাষা এবং বিবরণ র‍্যাঙ্কিংয়ে সাহায্য করে:
        - Android অ্যাপ: `android`, `mobile`, `apk` এর মতো টপিক।
        - Desktop অ্যাপ: `desktop`, `windows`, `linux`, `macos`, `compose-desktop`,
          `electron` এর মতো টপিক।
    - কিছু স্টার থাকলে Trending/Hot Release/Most Popular বিভাগে প্রদর্শিত হওয়ার সম্ভাবনা বাড়ে।

আপনার রিপোজিটরি এই শর্তগুলো পূরণ করলে, GitHub Store সার্চের মাধ্যমে এটি খুঁজে পেতে এবং
স্বয়ংক্রিয়ভাবে প্রদর্শন করতে পারে — কোনো ম্যানুয়াল জমা দেওয়ার প্রয়োজন নেই।

## 🧭 GitHub Store কিভাবে কাজ করে (সংক্ষিপ্ত)

1. **সার্চ**
    - প্ল্যাটফর্ম-অনুযায়ী কোয়েরি দিয়ে GitHub-এর `/search/repositories` এন্ডপয়েন্ট ব্যবহার করে।
    - টপিক, ভাষা এবং বিবরণের উপর ভিত্তি করে একটি সাধারণ স্কোরিং প্রয়োগ করে।
    - আর্কাইভ করা এবং কম সিগন্যালযুক্ত রিপোজিটরিগুলো ফিল্টার করে।

2. **রিলিজ ও ফাইল পরীক্ষা**
    - প্রার্থী রিপোজিটরিগুলোর জন্য `/repos/{owner}/{repo}/releases/latest` কল করে।
    - প্ল্যাটফর্ম-নির্দিষ্ট ফাইল এক্সটেনশনের জন্য `assets` অ্যারে পরীক্ষা করে।
    - কোনো উপযুক্ত ফাইল না পাওয়া গেলে, রিপোজিটরি ফলাফল থেকে বাদ দেওয়া হয়।
    - ব্যবহারকারীরা রিলিজ সিলেক্টরের মাধ্যমে সকল রিলিজও ব্রাউজ করতে পারেন।

3. **বিস্তারিত স্ক্রিন**
    - রিপোজিটরি তথ্য: নাম, মালিক, বিবরণ, স্টার, ফর্ক, ইস্যু।
    - রিলিজ ব্রাউজার: ট্যাগ, তারিখ, চেঞ্জলগ এবং ফাইলসহ যেকোনো রিলিজ ব্রাউজ করুন।
    - README: প্রধান শাখা থেকে লোড করে "এই অ্যাপ সম্পর্কে" হিসেবে রেন্ডার করা হয়।
    - ডেভেলপার প্রোফাইল লিংক এবং শেয়ার অ্যাকশন।
    - সরাসরি নেভিগেশনের জন্য ডিপ লিংকের মাধ্যমে প্রবেশযোগ্য।

4. **ইনস্টলেশন প্রক্রিয়া**
    - ব্যবহারকারী "সর্বশেষ সংস্করণ ইনস্টল করুন" ট্যাপ করলে বা একটি নির্দিষ্ট রিলিজ নির্বাচন করলে:
        - বর্তমান প্ল্যাটফর্মের জন্য সবচেয়ে উপযুক্ত ফাইল নির্বাচন করে (Android-এ আর্কিটেকচার ম্যাচিংসহ)।
        - ক্যাশ সাপোর্টসহ ডাউনলোড স্ট্রিম করে।
        - অপারেটিং সিস্টেমের ইনস্টলারে অর্পণ করে (Android-এ APK ইনস্টলার, Desktop-এ ডিফল্ট হ্যান্ডলার)।
        - Android-এ, একটি স্থানীয় ডাটাবেসে ইনস্টলেশন রেকর্ড করে এবং ইনস্টল করা অ্যাপের তালিকা সিঙ্ক রাখতে প্যাকেজ মনিটরিং ব্যবহার করে।
        - ম্যানেজড অ্যাপের জন্য খোলা, আনইনস্টল এবং ডাউনগ্রেড অ্যাকশন সমর্থন করে।

## ✅ সুবিধা / কেন GitHub Store ব্যবহার করবেন?

- **GitHub রিলিজে আর হাতড়ে বেড়ানো নয়**
  শুধুমাত্র সেই রিপোজিটরিগুলো দেখুন যেগুলো আসলে আপনার প্ল্যাটফর্মের জন্য বাইনারি বিতরণ করে।

- **আপনি কী ইনস্টল করেছেন তা জানে**
  GitHub Store-এর মাধ্যমে ইনস্টল করা অ্যাপ ট্র্যাক করে (Android) এবং নতুন রিলিজ উপলব্ধ হলে হাইলাইট করে, যাতে আপনি GitHub-এ আবার না খুঁজে আপডেট করতে পারেন।

- **সর্বদা আপডেট**
  ইনস্টলেশন ডিফল্টভাবে সর্বশেষ প্রকাশিত রিলিজ ব্যবহার করে, রিলিজ সিলেক্টরের মাধ্যমে
  যেকোনো পূর্ববর্তী রিলিজ থেকে ব্রাউজ ও ইনস্টল করার বিকল্পসহ।

- **সকল প্ল্যাটফর্মে অভিন্ন অভিজ্ঞতা**
  Android এবং Desktop-এর জন্য একই ইন্টারফেস ও লজিক, প্ল্যাটফর্ম-নেটিভ ইনস্টলেশন আচরণসহ।

- **ওপেন-সোর্স ও সম্প্রসারণযোগ্য**
  KMP-তে লেখা, নেটওয়ার্ক, ডোমেইন লজিক ও UI-এর মধ্যে পরিষ্কার বিভাজনসহ — সহজে ফর্ক,
  সম্প্রসারণ বা অভিযোজন করা যায়।

## 🔑 GitHub OAuth কনফিগারেশন

**সারসংক্ষেপ**
1. একটি GitHub OAuth App তৈরি করুন
2. **Client ID** কপি করুন
3. `local.properties`-এ রাখুন

<details>
<summary><strong>সম্পূর্ণ সেটআপ গাইড দেখুন</strong></summary>

  <br/>

### 1 - একটি GitHub OAuth App তৈরি করুন
যান:
**GitHub → Settings → Developer settings → OAuth Apps → New OAuth App**

| ক্ষেত্র                          | মান                                       |
| ------------------------------ | ------------------------------------------- |
| **Application name**           | আপনার পছন্দমতো (যেমন *GitHub Store Dev*) |
| **Homepage URL** | `https://github.com/username/repo_name`                   |
| **Authorization callback URL** | `githubstore://callback`                    |

তারপর **Create application**-এ ক্লিক করুন।

### 2 - আপনার Client ID কপি করুন
অ্যাপ তৈরির পর, GitHub দেখাবে:
- **Client ID** ← এটি আপনার প্রয়োজন
- **Client Secret** ← ❗ এই প্রকল্পের জন্য প্রয়োজন নেই

### 3 - আপনার প্রকল্পে যোগ করুন
আপনার প্রকল্পের `local.properties` ফাইল (প্রকল্পের মূল ডিরেক্টরি) খুলুন এবং যোগ করুন:
```properties
GITHUB_CLIENT_ID=YOUR_CLIENT_ID_HERE
```

### 4 - সিঙ্ক ও রান করুন
প্রকল্প সিঙ্ক করুন এবং অ্যাপ চালান। এখন আপনি GitHub দিয়ে লগ ইন করতে পারবেন।

### ❗ গুরুত্বপূর্ণ নোট
- `local.properties` **Git-এ আপলোড হয় না**, তাই আপনার Client ID স্থানীয় থাকে।
- এই প্রকল্পে শুধুমাত্র **Client ID** প্রয়োজন (Client Secret নয়)।
- প্রতিটি ডেভেলপারের উচিত ডেভেলপমেন্টের জন্য নিজের OAuth অ্যাপ তৈরি করা।

</details>
