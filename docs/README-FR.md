# 🗺️ Aperçu du projet

GitHub Store est une boutique d'applications multiplateforme pour les releases GitHub, conçue pour simplifier la découverte et l'installation de logiciels open source. Elle détecte automatiquement les binaires installables (APK, EXE, DMG, AppImage, DEB, RPM), offre une installation en un clic, suit les mises à jour et présente les informations des dépôts dans une interface épurée de type boutique d'applications.

Construite avec Kotlin Multiplatform et Compose Multiplatform pour les plateformes Android et Desktop.

</div>

> [!CAUTION]
> Android libre et open source est menacé. Google va transformer Android en une plateforme verrouillée, restreignant votre liberté essentielle d'installer les applications de votre choix. Faites entendre votre voix – [keepandroidopen.org](https://keepandroidopen.org/).

## 🚀 Fonctionnalités

- **Découverte intelligente**
    - Sections d'accueil pour les projets "Trending", "Hot Release" et "Most Popular" avec des filtres temporels.
    - Seuls les dépôts contenant des fichiers installables valides sont affichés.
    - Score de sujets adapté à la plateforme pour que les utilisateurs Android/bureau voient les apps pertinentes en premier.
    - Recherche améliorée avec un meilleur classement par pertinence et de meilleures performances.

- **Navigateur de versions et installations**
    - Sélecteur de versions pour explorer et installer depuis n'importe quelle release, pas seulement la plus récente.
    - Récupère toutes les releases de chaque dépôt.
    - Action unique "Installer la dernière version", plus une liste déroulante de toutes les releases disponibles et leurs installateurs.
    - Option d'installation manuelle avec vérifications de compatibilité automatiques.

- **Écran de détails enrichi**
    - Nom de l'app, version, bouton "Installer la dernière version" et action de partage.
    - Étoiles, forks, issues ouvertes.
    - Contenu du README rendu ("À propos de cette app").
    - Notes de release au format Markdown pour n'importe quelle release sélectionnée.
    - Liste des installateurs avec étiquettes de plateforme et tailles de fichier.
    - Prise en charge des liens profonds — ouvrez les détails du dépôt directement via URL.
    - Écran de profil du développeur pour explorer les dépôts et l'activité d'un développeur.

- **Gestion des applications**
    - Ouvrez, désinstallez et rétrogradez les applications installées directement depuis GitHub Store.
    - Android : correspondance d'architecture APK (armv7/armv8), surveillance des paquets et suivi des mises à jour.
    - Bureau (Windows/macOS/Linux) : télécharge les installateurs dans le dossier Téléchargements de l'utilisateur et les ouvre avec le gestionnaire par défaut.

- **Dépôts favoris**
    - Enregistrez et explorez vos dépôts favoris GitHub depuis l'application.

- **Réseau et performance**
    - Prise en charge du proxy dynamique pour un routage réseau configurable.
    - Système de cache amélioré pour un chargement plus rapide et une utilisation réduite de l'API.

- **Expérience utilisateur multiplateforme**
    - Android : écran de démarrage natif, gestion de l'expiration de session et icône adaptative.
    - Bureau : prise en charge d'AppImage sous Linux prioritaire, aux côtés des formats DEB et RPM.
    - Localisé en 12 langues : anglais, espagnol, français, japonais, coréen, polonais, russe, chinois, bengali, hindi, italien et turc.

## 🔍 Comment mon application apparaît-elle dans GitHub Store ?

GitHub Store n'utilise aucune indexation privée ni aucune règle de curation manuelle.
Votre projet peut apparaître automatiquement s'il remplit ces conditions :

1. **Dépôt public sur GitHub**
    - La visibilité doit être `public`.

2. **Fichiers installables dans la dernière release**
    - La dernière release doit contenir au moins un fichier avec une extension compatible :
        - Android : `.apk`
        - Windows : `.exe`, `.msi`
        - macOS : `.dmg`, `.pkg`
        - Linux : `.deb`, `.rpm`, `.AppImage`
    - GitHub Store ignore les artefacts de code source générés automatiquement (`Source code (zip)` /
      `Source code (tar.gz)`).

3. **Découvrable via la recherche / topics**
    - Les dépôts sont récupérés via l'API publique de recherche de GitHub.
    - Les topics, le langage et la description aident au classement :
        - Apps Android : topics comme `android`, `mobile`, `apk`.
        - Apps bureau : topics comme `desktop`, `windows`, `linux`, `macos`, `compose-desktop`,
          `electron`.
    - Avoir au moins quelques étoiles augmente la probabilité d'apparaître dans les sections Trending/Hot Release/Most Popular.

Si votre dépôt remplit ces conditions, GitHub Store peut le trouver via la recherche et l'afficher
automatiquement, sans soumission manuelle.

## 🧭 Comment fonctionne GitHub Store (vue d'ensemble)

1. **Recherche**
    - Utilise le endpoint `/search/repositories` de GitHub avec des requêtes adaptées à la plateforme.
    - Applique un score simple basé sur les topics, le langage et la description.
    - Filtre les dépôts archivés et ceux avec peu de signaux.

2. **Vérification des releases et des fichiers**
    - Pour les dépôts candidats, appelle `/repos/{owner}/{repo}/releases/latest`.
    - Vérifie le tableau `assets` à la recherche d'extensions de fichier spécifiques à la plateforme.
    - Si aucun fichier adapté n'est trouvé, le dépôt est exclu des résultats.
    - Les utilisateurs peuvent également explorer toutes les releases via le sélecteur de versions.

3. **Écran de détails**
    - Informations du dépôt : nom, propriétaire, description, étoiles, forks, issues.
    - Explorateur de releases : parcourez n'importe quelle release avec son tag, sa date, son changelog et ses fichiers.
    - README : chargé depuis la branche principale et rendu comme "À propos de cette app".
    - Lien vers le profil du développeur et action de partage.
    - Accessible via les liens profonds pour une navigation directe.

4. **Processus d'installation**
    - Lorsque l'utilisateur appuie sur "Installer la dernière version" ou sélectionne une release spécifique :
        - Sélectionne le fichier le plus adapté à la plateforme actuelle (avec correspondance d'architecture sur Android).
        - Diffuse le téléchargement avec prise en charge du cache.
        - Délègue à l'installateur du système d'exploitation (installateur APK sur Android, gestionnaire par défaut sur bureau).
        - Sur Android, enregistre l'installation dans une base de données locale et utilise la surveillance des paquets pour maintenir la liste des apps installées synchronisée.
        - Prend en charge les actions d'ouverture, de désinstallation et de rétrogradation pour les apps gérées.

## ✅ Avantages / Pourquoi utiliser GitHub Store ?

- **Plus besoin de fouiller dans les releases GitHub**
  Ne voyez que les dépôts qui distribuent réellement des binaires pour votre plateforme.

- **Sait ce que vous avez installé**
  Suit les apps installées via GitHub Store (Android) et signale quand de nouvelles releases sont disponibles, pour que vous puissiez mettre à jour sans chercher de nouveau sur GitHub.

- **Toujours à jour**
  Les installations utilisent par défaut la dernière release publiée, avec la possibilité d'explorer et d'installer depuis
  n'importe quelle release antérieure via le sélecteur de versions.

- **Expérience uniforme sur toutes les plateformes**
  La même interface et la même logique pour Android et bureau, avec un comportement d'installation natif de la plateforme.

- **Open source et extensible**
  Écrit en KMP avec une séparation claire entre réseau, logique de domaine et interface utilisateur — facile à forker,
  étendre ou adapter.

## 🔑 Configuration OAuth GitHub

**Résumé**
1. Créez une GitHub OAuth App
2. Copiez le **Client ID**
3. Ajoutez-le dans `local.properties`

<details>
<summary><strong>Afficher le guide de configuration complet</strong></summary>

  <br/>

### 1 - Créer une GitHub OAuth App
Allez sur :
**GitHub → Settings → Developer settings → OAuth Apps → New OAuth App**

| Champ                          | Valeur                                       |
| ------------------------------ | ------------------------------------------- |
| **Application name**           | Ce que vous préférez (ex. *GitHub Store Dev*) |
| **Homepage URL** | `https://github.com/username/repo_name`                   |
| **Authorization callback URL** | `githubstore://callback`                    |

Puis cliquez sur **Create application**.

### 2 - Copier votre Client ID
Après avoir créé l'app, GitHub affichera :
- **Client ID** ← c'est ce dont vous avez besoin
- **Client Secret** ← ❗ NON requis pour ce projet

### 3 - L'ajouter à votre projet
Ouvrez le fichier `local.properties` de votre projet (racine du projet) et ajoutez :
```properties
GITHUB_CLIENT_ID=YOUR_CLIENT_ID_HERE
```

### 4 - Synchroniser et exécuter
Synchronisez le projet et lancez l'app. Vous devriez maintenant pouvoir vous connecter avec GitHub.

### ❗ Notes importantes
- `local.properties` **n'est pas envoyé sur Git**, votre Client ID reste donc local.
- Ce projet nécessite uniquement le **Client ID** (pas le Client Secret).
- Chaque développeur devrait créer sa propre OAuth app pour le développement.

</details>
