# 🗺️ परियोजना अवलोकन

GitHub Store, GitHub रिलीज़ के लिए एक क्रॉस-प्लेटफ़ॉर्म ऐप स्टोर है, जिसे ओपन-सोर्स सॉफ़्टवेयर की खोज और इंस्टॉलेशन को सरल बनाने के लिए डिज़ाइन किया गया है। यह स्वचालित रूप से इंस्टॉल करने योग्य बाइनरी (APK, EXE, DMG, AppImage, DEB, RPM) का पता लगाता है, एक क्लिक में इंस्टॉलेशन प्रदान करता है, अपडेट ट्रैक करता है और रिपॉज़िटरी की जानकारी को एक साफ़ ऐप स्टोर जैसे इंटरफ़ेस में प्रस्तुत करता है।

Kotlin Multiplatform और Compose Multiplatform के साथ Android और Desktop प्लेटफ़ॉर्म के लिए निर्मित।

## 🚀 विशेषताएँ

- **स्मार्ट खोज**
    - "Trending", "Hot Release" और "Most Popular" प्रोजेक्ट के लिए समय-आधारित फ़िल्टर के साथ होम सेक्शन।
    - केवल मान्य इंस्टॉल करने योग्य फ़ाइलों वाले रिपॉज़िटरी दिखाए जाते हैं।
    - प्लेटफ़ॉर्म-जागरूक टॉपिक स्कोरिंग ताकि Android/Desktop उपयोगकर्ता पहले प्रासंगिक ऐप्स देखें।
    - बेहतर प्रासंगिकता रैंकिंग और प्रदर्शन के साथ नवीनीकृत खोज।

- **रिलीज़ ब्राउज़र और इंस्टॉलेशन**
    - किसी भी रिलीज़ से ब्राउज़ और इंस्टॉल करने के लिए रिलीज़ सेलेक्टर, केवल नवीनतम तक सीमित नहीं।
    - प्रत्येक रिपॉज़िटरी की सभी रिलीज़ प्राप्त करता है।
    - एकल "नवीनतम इंस्टॉल करें" क्रिया, साथ ही सभी उपलब्ध रिलीज़ और उनके इंस्टॉलर की ड्रॉपडाउन सूची।
    - स्वचालित संगतता जाँच के साथ मैनुअल इंस्टॉलेशन विकल्प।

- **विस्तृत विवरण स्क्रीन**
    - ऐप का नाम, संस्करण, "नवीनतम इंस्टॉल करें" बटन और शेयर क्रिया।
    - स्टार्स, फ़ोर्क्स, खुले इश्यू।
    - रेंडर किया गया README कंटेंट ("इस ऐप के बारे में")।
    - किसी भी चयनित रिलीज़ के लिए Markdown-स्वरूपित रिलीज़ नोट्स।
    - प्लेटफ़ॉर्म टैग और फ़ाइल आकार के साथ इंस्टॉलर सूची।
    - डीप लिंक समर्थन — URL के माध्यम से सीधे रिपॉज़िटरी विवरण खोलें।
    - डेवलपर के रिपॉज़िटरी और गतिविधि का अन्वेषण करने के लिए डेवलपर प्रोफ़ाइल स्क्रीन।

- **ऐप प्रबंधन**
    - GitHub Store से सीधे इंस्टॉल किए गए ऐप्स खोलें, अनइंस्टॉल करें और डाउनग्रेड करें।
    - Android: APK आर्किटेक्चर मैचिंग (armv7/armv8), पैकेज मॉनिटरिंग और अपडेट ट्रैकिंग।
    - Desktop (Windows/macOS/Linux): इंस्टॉलर को उपयोगकर्ता के Downloads फ़ोल्डर में डाउनलोड करता है और डिफ़ॉल्ट हैंडलर से खोलता है।

- **तारांकित रिपॉज़िटरी**
    - ऐप से अपने तारांकित GitHub रिपॉज़िटरी सहेजें और ब्राउज़ करें।

- **नेटवर्क और प्रदर्शन**
    - कॉन्फ़िगर करने योग्य नेटवर्क रूटिंग के लिए डायनामिक प्रॉक्सी समर्थन।
    - तेज़ लोडिंग और कम API उपयोग के लिए बेहतर कैश सिस्टम।

- **क्रॉस-प्लेटफ़ॉर्म UX**
    - Android: नेटिव स्प्लैश स्क्रीन, सत्र समाप्ति हैंडलिंग और एडेप्टिव आइकन।

    - 12 भाषाओं में स्थानीयकृत: अंग्रेज़ी, स्पैनिश, फ़्रेंच, जापानी, कोरियाई, पोलिश, रूसी, चीनी, बंगाली, हिन्दी, इतालवी और तुर्की।

## 🔍 मेरा ऐप GitHub Store में कैसे दिखाई देता है?

GitHub Store किसी भी प्रकार की निजी इंडेक्सिंग या मैनुअल क्यूरेशन नियमों का उपयोग नहीं करता।
यदि आपका प्रोजेक्ट इन शर्तों को पूरा करता है तो वह स्वचालित रूप से दिखाई दे सकता है:

1. **GitHub पर सार्वजनिक रिपॉज़िटरी**
    - दृश्यता `public` होनी चाहिए।

2. **नवीनतम रिलीज़ में इंस्टॉल करने योग्य फ़ाइलें**
    - नवीनतम रिलीज़ में कम से कम एक संगत एक्सटेंशन वाली फ़ाइल होनी चाहिए:
        - Android: `.apk`
        - Windows: `.exe`, `.msi`
        - macOS: `.dmg`, `.pkg`
        - Linux: `.deb`, `.rpm`, `.AppImage`
    - GitHub Store स्वचालित रूप से जनरेट किए गए सोर्स कोड आर्टिफ़ैक्ट (`Source code (zip)` /
      `Source code (tar.gz)`) को अनदेखा करता है।

3. **खोज / टॉपिक के माध्यम से खोज योग्य**
    - रिपॉज़िटरी GitHub की सार्वजनिक खोज API के माध्यम से प्राप्त की जाती हैं।
    - टॉपिक, भाषा और विवरण रैंकिंग में सहायता करते हैं:
        - Android ऐप्स: `android`, `mobile`, `apk` जैसे टॉपिक।
        - Desktop ऐप्स: `desktop`, `windows`, `linux`, `macos`, `compose-desktop`,
          `electron` जैसे टॉपिक।
    - कम से कम कुछ स्टार्स होने से Trending/Hot Release/Most Popular सेक्शन में दिखने की संभावना बढ़ती है।

यदि आपकी रिपॉज़िटरी इन शर्तों को पूरा करती है, तो GitHub Store इसे खोज के माध्यम से ढूंढ सकता है और
स्वचालित रूप से प्रदर्शित कर सकता है — किसी मैनुअल सबमिशन की आवश्यकता नहीं।

## 🧭 GitHub Store कैसे काम करता है (अवलोकन)

1. **खोज**
    - प्लेटफ़ॉर्म-अनुकूलित क्वेरी के साथ GitHub के `/search/repositories` एंडपॉइंट का उपयोग करता है।
    - टॉपिक, भाषा और विवरण पर आधारित एक सरल स्कोरिंग लागू करता है।
    - आर्काइव किए गए और कम सिग्नल वाले रिपॉज़िटरी को फ़िल्टर करता है।

2. **रिलीज़ और फ़ाइल जाँच**
    - उम्मीदवार रिपॉज़िटरी के लिए `/repos/{owner}/{repo}/releases/latest` को कॉल करता है।
    - प्लेटफ़ॉर्म-विशिष्ट फ़ाइल एक्सटेंशन के लिए `assets` ऐरे की जाँच करता है।
    - यदि कोई उपयुक्त फ़ाइल नहीं मिलती, तो रिपॉज़िटरी को परिणामों से बाहर कर दिया जाता है।
    - उपयोगकर्ता रिलीज़ सेलेक्टर के माध्यम से सभी रिलीज़ भी ब्राउज़ कर सकते हैं।

3. **विवरण स्क्रीन**
    - रिपॉज़िटरी जानकारी: नाम, मालिक, विवरण, स्टार्स, फ़ोर्क्स, इश्यू।
    - रिलीज़ एक्सप्लोरर: किसी भी रिलीज़ को उसके टैग, दिनांक, चेंजलॉग और फ़ाइलों के साथ ब्राउज़ करें।
    - README: मुख्य ब्रांच से लोड किया गया और "इस ऐप के बारे में" के रूप में रेंडर किया गया।
    - डेवलपर प्रोफ़ाइल लिंक और शेयर क्रिया।
    - सीधी नेविगेशन के लिए डीप लिंक के माध्यम से पहुँच योग्य।

4. **इंस्टॉलेशन प्रक्रिया**
    - जब उपयोगकर्ता "नवीनतम इंस्टॉल करें" पर टैप करता है या कोई विशिष्ट रिलीज़ चुनता है:
        - वर्तमान प्लेटफ़ॉर्म के लिए सबसे उपयुक्त फ़ाइल चुनता है (Android पर आर्किटेक्चर मैचिंग के साथ)।
        - कैश समर्थन के साथ डाउनलोड स्ट्रीम करता है।
        - ऑपरेटिंग सिस्टम इंस्टॉलर को सौंपता है (Android पर APK इंस्टॉलर, Desktop पर डिफ़ॉल्ट हैंडलर)।
        - Android पर, इंस्टॉलेशन को स्थानीय डेटाबेस में पंजीकृत करता है और इंस्टॉल किए गए ऐप्स की सूची को सिंक रखने के लिए पैकेज मॉनिटरिंग का उपयोग करता है।
        - प्रबंधित ऐप्स के लिए खोलने, अनइंस्टॉल करने और डाउनग्रेड करने की क्रियाओं का समर्थन करता है।

## ✅ फायदे / GitHub Store क्यों इस्तेमाल करें?

- **GitHub रिलीज़ में खोजबीन की ज़रूरत नहीं**
  केवल वे रिपॉज़िटरी देखें जो वास्तव में आपके प्लेटफ़ॉर्म के लिए बाइनरी वितरित करती हैं।

- **जानता है कि आपने क्या इंस्टॉल किया**
  GitHub Store (Android) के माध्यम से इंस्टॉल किए गए ऐप्स को ट्रैक करता है और जब नई रिलीज़ उपलब्ध होती हैं तो हाइलाइट करता है, ताकि आप GitHub पर दोबारा खोजे बिना अपडेट कर सकें।

- **हमेशा अपडेट**
  इंस्टॉलेशन डिफ़ॉल्ट रूप से नवीनतम प्रकाशित रिलीज़ का उपयोग करते हैं, रिलीज़ सेलेक्टर के माध्यम से
  किसी भी पिछली रिलीज़ से ब्राउज़ और इंस्टॉल करने के विकल्प के साथ।

- **सभी प्लेटफ़ॉर्म पर एकसमान अनुभव**
  Android और Desktop के लिए समान इंटरफ़ेस और लॉजिक, प्लेटफ़ॉर्म-नेटिव इंस्टॉलेशन व्यवहार के साथ।

- **ओपन-सोर्स और विस्तार योग्य**
  नेटवर्क, डोमेन लॉजिक और UI के बीच स्पष्ट विभाजन के साथ KMP में लिखा गया — फ़ोर्क, विस्तार या
  अनुकूलन करना आसान।

## 🔑 GitHub OAuth कॉन्फ़िगरेशन

**सारांश**
1. एक GitHub OAuth App बनाएँ
2. **Client ID** कॉपी करें
3. इसे `local.properties` में डालें

<details>
<summary><strong>पूर्ण सेटअप गाइड दिखाएँ</strong></summary>

  <br/>

### 1 - GitHub OAuth App बनाएँ
यहाँ जाएँ:
**GitHub → Settings → Developer settings → OAuth Apps → New OAuth App**

| फ़ील्ड                          | मान                                       |
| ------------------------------ | ------------------------------------------- |
| **Application name**           | आप जो चाहें (उदा. *GitHub Store Dev*) |
| **Homepage URL** | `https://github.com/username/repo_name`                   |
| **Authorization callback URL** | `githubstore://callback`                    |

फिर **Create application** पर क्लिक करें।

### 2 - अपनी Client ID कॉपी करें
ऐप बनाने के बाद, GitHub दिखाएगा:
- **Client ID** ← यही आपको चाहिए
- **Client Secret** ← ❗ इस प्रोजेक्ट के लिए आवश्यक नहीं

### 3 - इसे अपने प्रोजेक्ट में जोड़ें
अपने प्रोजेक्ट की `local.properties` फ़ाइल (प्रोजेक्ट रूट) खोलें और जोड़ें:
```properties
GITHUB_CLIENT_ID=YOUR_CLIENT_ID_HERE
```

### 4 - सिंक और रन करें
प्रोजेक्ट को सिंक करें और ऐप चलाएँ। अब आप GitHub से लॉगिन कर सकते हैं।

### ❗ महत्वपूर्ण नोट्स
- `local.properties` **Git में अपलोड नहीं होती**, इसलिए आपकी Client ID स्थानीय रहती है।
- इस प्रोजेक्ट को केवल **Client ID** की आवश्यकता है (Client Secret नहीं)।
- प्रत्येक डेवलपर को विकास के लिए अपनी OAuth app बनानी चाहिए।

</details>

## ⚠️ अस्वीकरण

GitHub Store केवल उन रिलीज़ फ़ाइलों को खोजने और डाउनलोड करने में आपकी मदद करता है जो पहले से
बाहरी डेवलपर्स द्वारा GitHub पर प्रकाशित हैं।
उन डाउनलोड की सामग्री, सुरक्षा और व्यवहार पूरी तरह से उनके
संबंधित लेखकों और वितरकों की ज़िम्मेदारी है, इस प्रोजेक्ट की नहीं।

GitHub Store का उपयोग करके, आप समझते और स्वीकार करते हैं कि आप किसी भी डाउनलोड किए गए सॉफ़्टवेयर को
अपनी ज़िम्मेदारी पर इंस्टॉल और चलाते हैं।
यह प्रोजेक्ट किसी भी इंस्टॉलर की समीक्षा, सत्यापन या गारंटी नहीं देता कि वह सुरक्षित है, मैलवेयर-मुक्त है या
किसी विशेष उद्देश्य के लिए उपयुक्त है।
