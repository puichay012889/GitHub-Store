# 🗺️ 프로젝트 개요

GitHub Store는 GitHub 릴리스를 위한 크로스 플랫폼 앱 스토어로, 오픈소스 소프트웨어의 발견과 설치를 간소화하도록 설계되었습니다. 설치 가능한 바이너리(APK, EXE, DMG, AppImage, DEB, RPM)를 자동으로 감지하고, 원클릭 설치를 제공하며, 업데이트를 추적하고, 리포지토리 정보를 깔끔한 앱 스토어 스타일의 인터페이스로 제공합니다.

Kotlin Multiplatform과 Compose Multiplatform으로 구축되어 Android 및 Desktop 플랫폼을 지원합니다.

</div>

> [!CAUTION]
> 무료 오픈소스 Android가 위협받고 있습니다. Google은 Android를 폐쇄적인 플랫폼으로 전환하여, 원하는 앱을 설치할 수 있는 필수적인 자유를 제한하려 합니다. 여러분의 목소리를 내주세요 – [keepandroidopen.org](https://keepandroidopen.org/).

- **스마트 발견**
    - "Trending", "Hot Release", "Most Popular" 프로젝트를 시간 기반 필터와 함께 홈 섹션에서 제공합니다.
    - 유효한 설치 가능 파일이 있는 리포지토리만 표시됩니다.
    - 플랫폼 인식 토픽 점수 매김으로 Android/데스크톱 사용자가 관련 앱을 먼저 볼 수 있습니다.
    - 향상된 관련성 순위와 성능을 갖춘 개선된 검색 기능.

- **릴리스 브라우저 및 설치**
    - 최신 릴리스뿐만 아니라 모든 릴리스를 탐색하고 설치할 수 있는 릴리스 선택기.
    - 각 리포지토리의 모든 릴리스를 가져옵니다.
    - 단일 "최신 버전 설치" 액션과 모든 사용 가능한 릴리스 및 설치 파일의 드롭다운 목록.
    - 자동 호환성 검사가 포함된 수동 설치 옵션.

- **상세 정보 화면**
    - 앱 이름, 버전, "최신 버전 설치" 버튼 및 공유 액션.
    - 스타, 포크, 열린 이슈.
    - 렌더링된 README 콘텐츠 ("이 앱 소개").
    - 선택된 릴리스에 대한 Markdown 형식의 릴리스 노트.
    - 플랫폼 태그 및 파일 크기가 포함된 설치 파일 목록.
    - 딥 링크 지원 — URL을 통해 리포지토리 상세 정보를 직접 열 수 있습니다.
    - 개발자의 리포지토리와 활동을 탐색할 수 있는 개발자 프로필 화면.

- **앱 관리**
    - GitHub Store에서 직접 설치된 앱을 열고, 삭제하고, 다운그레이드할 수 있습니다.
    - Android: APK 아키텍처 매칭(armv7/armv8), 패키지 모니터링 및 업데이트 추적.

- **스타 리포지토리**
    - 앱 내에서 GitHub 스타 리포지토리를 저장하고 탐색할 수 있습니다.

- **네트워크 및 성능**
    - 구성 가능한 네트워크 라우팅을 위한 동적 프록시 지원.

## 🔍 내 앱은 어떻게 GitHub Store에 표시되나요?

GitHub Store는 비공개 인덱싱이나 수동 큐레이션 규칙을 사용하지 않습니다.
다음 조건을 충족하면 프로젝트가 자동으로 표시될 수 있습니다:

1. **공개 GitHub 리포지토리**
    - 가시성이 `public`이어야 합니다.

2. **최신 릴리스에 설치 가능한 파일 포함**
    - 최신 릴리스에 호환되는 확장자를 가진 파일이 최소 하나 포함되어야 합니다:
        - Android: `.apk`
        - Windows: `.exe`, `.msi`
        - macOS: `.dmg`, `.pkg`
        - Linux: `.deb`, `.rpm`, `.AppImage`
    - GitHub Store는 자동 생성된 소스 코드 아티팩트(`Source code (zip)` /
      `Source code (tar.gz)`)를 무시합니다.

3. **검색 / 토픽을 통한 발견 가능성**
    - 리포지토리는 GitHub의 공개 검색 API를 통해 가져옵니다.
    - 토픽, 언어, 설명이 순위에 도움됩니다:
        - Android 앱: `android`, `mobile`, `apk`와 같은 토픽.
        - 데스크톱 앱: `desktop`, `windows`, `linux`, `macos`, `compose-desktop`,
          `electron`과 같은 토픽.
    - 최소한의 스타가 있으면 Trending/Hot Release/Most Popular 섹션에 표시될 확률이 높아집니다.

리포지토리가 이러한 조건을 충족하면, GitHub Store는 검색을 통해 자동으로 찾아 표시할 수 있으며,
수동 제출이 필요하지 않습니다.

## 🧭 GitHub Store 작동 방식 (개요)

1. **검색**
    - 플랫폼에 맞게 조정된 쿼리로 GitHub의 `/search/repositories` 엔드포인트를 사용합니다.
    - 토픽, 언어, 설명을 기반으로 간단한 점수를 적용합니다.
    - 아카이브된 리포지토리와 신호가 적은 리포지토리를 필터링합니다.

2. **릴리스 및 파일 확인**
    - 후보 리포지토리에 대해 `/repos/{owner}/{repo}/releases/latest`를 호출합니다.
    - `assets` 배열에서 플랫폼별 파일 확장자를 확인합니다.
    - 적합한 파일이 없으면 해당 리포지토리는 결과에서 제외됩니다.
    - 사용자는 릴리스 선택기를 통해 모든 릴리스를 탐색할 수도 있습니다.

3. **상세 정보 화면**
    - 리포지토리 정보: 이름, 소유자, 설명, 스타, 포크, 이슈.
    - 릴리스 탐색기: 태그, 날짜, 변경 로그 및 파일과 함께 모든 릴리스를 탐색합니다.
    - README: 메인 브랜치에서 로드되어 "이 앱 소개"로 렌더링됩니다.
    - 개발자 프로필 링크 및 공유 액션.
    - 직접 탐색을 위한 딥 링크로 접근 가능합니다.

4. **설치 과정**
    - 사용자가 "최신 버전 설치"를 누르거나 특정 릴리스를 선택하면:
        - 현재 플랫폼에 가장 적합한 파일을 선택합니다 (Android에서는 아키텍처 매칭 포함).
        - 캐시 지원과 함께 다운로드를 스트리밍합니다.
        - OS 설치 프로그램에 위임합니다 (Android에서는 APK 설치 프로그램, 데스크톱에서는 기본 핸들러).
        - Android에서는 설치를 로컬 데이터베이스에 등록하고 패키지 모니터링을 사용하여 설치된 앱 목록을 동기화 상태로 유지합니다.
        - 관리되는 앱에 대한 열기, 삭제, 다운그레이드 액션을 지원합니다.

## ✅ 장점 / 왜 GitHub Store를 사용하나요?

- **GitHub 릴리스를 더 이상 뒤질 필요 없습니다**
  실제로 해당 플랫폼용 바이너리를 배포하는 리포지토리만 표시됩니다.

- **설치한 것을 기억합니다**
  GitHub Store를 통해 설치된 앱을 추적하고 (Android), 새로운 릴리스가 있을 때 강조 표시하여 GitHub를 다시 검색하지 않고도 업데이트할 수 있습니다.

- **항상 최신 상태**
  설치는 기본적으로 최신 릴리스를 사용하며, 릴리스 선택기를 통해 이전 릴리스를 탐색하고 설치할 수 있는 옵션이 있습니다.

- **모든 플랫폼에서 일관된 경험**
  Android와 데스크톱에서 동일한 인터페이스와 로직을 사용하며, 플랫폼 네이티브 설치 동작을 제공합니다.

- **오픈소스이며 확장 가능**
  네트워크, 도메인 로직, UI 간의 명확한 분리로 KMP로 작성되어 포크, 확장 또는 적응이 쉽습니다.

## 🔑 GitHub OAuth 설정

**요약**
1. GitHub OAuth App 생성
2. **Client ID** 복사
3. `local.properties`에 추가

<details>
<summary><strong>전체 설정 가이드 보기</strong></summary>

  <br/>

### 1 - GitHub OAuth App 생성
이동:
**GitHub → Settings → Developer settings → OAuth Apps → New OAuth App**

| 필드                             | 값                                          |
| ------------------------------ | ------------------------------------------- |
| **Application name**           | 원하는 이름 (예: *GitHub Store Dev*)         |
| **Homepage URL** | `https://github.com/username/repo_name`                   |
| **Authorization callback URL** | `githubstore://callback`                    |

그런 다음 **Create application**을 클릭합니다.

### 2 - Client ID 복사
앱 생성 후 GitHub에 표시됩니다:
- **Client ID** ← 이것이 필요합니다
- **Client Secret** ← ❗ 이 프로젝트에는 필요하지 않습니다

### 3 - 프로젝트에 추가
프로젝트의 `local.properties` 파일(프로젝트 루트)을 열고 다음을 추가합니다:
```properties
GITHUB_CLIENT_ID=YOUR_CLIENT_ID_HERE
```

### 4 - 동기화 및 실행
프로젝트를 동기화하고 앱을 실행합니다. 이제 GitHub로 로그인할 수 있어야 합니다.

### ❗ 중요 사항
- `local.properties`는 **Git에 커밋되지 않으므로** Client ID가 로컬에 유지됩니다.
- 이 프로젝트는 **Client ID**만 필요합니다 (Client Secret은 불필요).
- 각 개발자는 개발용 자체 OAuth App을 생성해야 합니다.

</details>

## ⚠️ 면책 조항

GitHub Store는 외부 개발자가 GitHub에 이미 게시한 릴리스 파일을 발견하고 다운로드하는 것만
도와줍니다.
해당 다운로드의 콘텐츠, 보안 및 동작은 이 프로젝트가 아닌 해당 저작자 및 배포자의 전적인
책임입니다.

GitHub Store를 사용함으로써, 다운로드한 소프트웨어의 설치 및 실행은 본인의 책임 하에
이루어진다는 것을 이해하고 동의합니다.
이 프로젝트는 설치 프로그램이 안전하거나 멀웨어가 없거나 특정 목적에 적합한지 검토, 검증 또는
보증하지 않습니다.
