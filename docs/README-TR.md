# 🗺️ Proje Genel Bakisi

GitHub Store, acik kaynak yazilimlarin kesfini ve kurulumunu kolaylastirmak icin tasarlanmis, GitHub surumleri icin capraz platformlu bir uygulama magazasidir. Kurulabilir ikili dosyalari (APK, EXE, DMG, AppImage, DEB, RPM) otomatik olarak algilar, tek tikla kurulum sunar, guncellemeleri takip eder ve depo bilgilerini temiz bir uygulama magazasi arayuzunde sunar.

Kotlin Multiplatform ve Compose Multiplatform ile Android ve Masaustu platformlari icin gelistirilmistir.

## 🚀 Ozellikler

- **Akilli kesif**
    - "Trending", "Hot Release" ve "Most Popular" projeleri icin zamana dayali filtrelerle ana sayfa bolumleri.
    - Yalnizca gecerli kurulabilir dosyalara sahip depolar gosterilir.
    - Android/masaustu kullanicilarinin ilgili uygulamalari once gormesini saglayan platforma duyarli konu puanlamasi.
    - Daha iyi alaka siralama ve performansa sahip yenilenenmis arama.

- **Surum tarayicisi ve kurulum**
    - Yalnizca en son surum degil, herhangi bir surumden kesif ve kurulum icin surum secici.
    - Her deponun tum surumlerini getirir.
    - Tek tusla "En son surumu kur" eylemi, artı tum mevcut surumler ve yukleyicilerin acilir listesi.
    - Otomatik uyumluluk kontrolleriyle manuel kurulum secenegi.

- **Zengin detay ekrani**
    - Uygulama adi, surum, "En son surumu kur" butonu ve paylasim eylemi.
    - Yildizlar, fork'lar, acik issue'lar.
    - Islenmis README icerigi ("Bu uygulama hakkinda").
    - Secilen herhangi bir surum icin Markdown formatli surum notlari.
    - Platform etiketleri ve dosya boyutlariyla yukleyici listesi.
    - Derin baglanti destegi — URL araciligiyla dogrudan depo detaylarina eris.
    - Bir gelistiricinin depolarini ve etkinliklerini kesfetmek icin gelistirici profil ekrani.

- **Uygulama yonetimi**
    - Kurulu uygulamalari dogrudan GitHub Store uzerinden ac, kaldir ve eski surume geri al.
    - Android: APK mimari eslestirmesi (armv7/armv8), paket izleme ve guncelleme takibi.
    - Masaustu (Windows/macOS/Linux): yukleyicileri kullanicinin Indirilenler klasorune indirir ve varsayilan isleyiciyle acar.

- **Yildizli depolar**
    - GitHub'da yildizladiginiz depolari uygulama icinden kaydet ve kesfet.

- **Ag ve performans**
    - Yapilandirmaya dayali ag yonlendirmesi icin dinamik proxy destegi.
    - Daha hizli yukleme ve daha az API kullanimi icin gelistirilmis onbellek sistemi.

- **Capraz platform UX**
    - Android: yerel acilis ekrani, oturum suresi dolumu yonetimi ve uyarlanabilir simge.
    - Masaustu: Linux'ta DEB ve RPM formatlariyla birlikte oncelikli AppImage destegi.
    - 12 dilde yerellestirilmistir: Ingilizce, Ispanyolca, Fransizca, Japonca, Korece, Lehce, Rusca, Cince, Bengalce, Hintce, Italyanca ve Turkce.

## 🔍 Uygulamam GitHub Store'da nasil gorunur?

GitHub Store hicbir ozel indeksleme veya manuel kurasyon kurali kullanmaz.
Projeniz asagidaki kosullari karsiliyorsa otomatik olarak gorunur hale gelebilir:

1. **GitHub'da herkese acik depo**
    - Gorunurluk `public` olmalidir.

2. **En son surumde kurulabilir dosyalar**
    - En son surum, uyumlu bir uzantiya sahip en az bir dosya icermelidir:
        - Android: `.apk`
        - Windows: `.exe`, `.msi`
        - macOS: `.dmg`, `.pkg`
        - Linux: `.deb`, `.rpm`, `.AppImage`
    - GitHub Store, otomatik olusturulan kaynak kodu artifaktlarini (`Source code (zip)` /
      `Source code (tar.gz)`) yok sayar.

3. **Arama / konular araciligiyla kesfedilebilirlik**
    - Depolar, GitHub'in herkese acik arama API'si araciligiyla elde edilir.
    - Konular, dil ve aciklama siralamada yardimci olur:
        - Android uygulamalari: `android`, `mobile`, `apk` gibi konular.
        - Masaustu uygulamalari: `desktop`, `windows`, `linux`, `macos`, `compose-desktop`,
          `electron` gibi konular.
    - En az birkas yildiza sahip olmak, Trending/Hot Release/Most Popular bolumlerinde gorunme olasiliginizi artirir.

Deponuz bu kosullari karsiliyorsa, GitHub Store onu arama yoluyla bulabilir ve
otomatik olarak goruntuleyebilir — manuel gonderime gerek yoktur.

## 🧭 GitHub Store nasil calisir (genel bakis)

1. **Arama**
    - GitHub'in `/search/repositories` uç noktasini platforma uyarlanmis sorgularla kullanir.
    - Konular, dil ve aciklamaya dayali basit bir puanlama uygular.
    - Arsivlenmis ve dusuk sinyalli depolari filtreler.

2. **Surum ve dosya kontrolu**
    - Aday depolar icin `/repos/{owner}/{repo}/releases/latest` cagrisini yapar.
    - `assets` dizisini platforma ozgu dosya uzantilari acisindan dogrular.
    - Uygun dosya bulunamazsa, depo sonuclardan cikarilir.
    - Kullanicilar ayrica surum secici araciligiyla tum surumleri kesfedebilir.

3. **Detay ekrani**
    - Depo bilgileri: ad, sahip, aciklama, yildizlar, fork'lar, issue'lar.
    - Surum gezgini: etiketi, tarihi, degisiklik gunlugu ve dosyalariyla herhangi bir surumu gezin.
    - README: ana daldan yuklenir ve "Bu uygulama hakkinda" olarak islenir.
    - Gelistirici profil baglantisi ve paylasim eylemi.
    - Dogrudan gezinme icin derin baglantilarla erisilebilir.

4. **Kurulum sureci**
    - Kullanici "En son surumu kur"a dokundigunda veya belirli bir surum sectiginde:
        - Mevcut platform icin en uygun dosyayi secer (Android'de mimari eslestirmesi ile).
        - Onbellek destegi ile indirmeyi akis olarak iletir.
        - Isletim sistemi yukleyicisine devreder (Android'de APK yukleyici, masaustunde varsayilan isleyici).
        - Android'de kurulumu yerel bir veritabanina kaydeder ve kurulu uygulamalar listesini senkronize tutmak icin paket izlemeyi kullanir.
        - Yonetilen uygulamalar icin acma, kaldirma ve eski surume geri alma eylemlerini destekler.

## ✅ Avantajlar / Neden GitHub Store kullanmaliyim?

- **GitHub surumlerinde arama yapmaya son**
  Yalnizca platformunuz icin gercekten ikili dosya dagitan depolari gorun.

- **Ne kurdugunuzu bilir**
  GitHub Store araciligiyla kurulan uygulamalari takip eder (Android) ve yeni surumler mevcut oldugunda vurgular, boylece GitHub'da tekrar aramadan guncelleyebilirsiniz.

- **Her zaman guncel**
  Kurulumlar varsayilan olarak en son yayinlanan surumu kullanir, surum secici araciligiyla herhangi bir onceki surumden kesif ve kurulum secenegi ile.

- **Tum platformlarda tutarli deneyim**
  Android ve masaustu icin ayni arayuz ve mantik, platforma ozgu yerel kurulum davranisi ile.

- **Acik kaynakli ve genisletilebilir**
  Ag, alan mantigi ve kullanici arayuzu arasinda net bir ayrimla KMP ile yazilmistir — kolayca fork'lanabilir,
  genisletilebilir veya uyarlanabilir.

## 🔑 GitHub OAuth Yapilandirmasi

**Ozet**
1. Bir GitHub OAuth Uygulamasi olusturun
2. **Client ID**'yi kopyalayin
3. `local.properties` dosyasina ekleyin

<details>
<summary><strong>Tam yapilandirma rehberini goster</strong></summary>

  <br/>

### 1 - GitHub OAuth Uygulamasi Olusturun
Su adrese gidin:
**GitHub → Settings → Developer settings → OAuth Apps → New OAuth App**

| Alan                           | Deger                                       |
| ------------------------------ | ------------------------------------------- |
| **Application name**           | Istediginiz bir sey (ornegin *GitHub Store Dev*) |
| **Homepage URL** | `https://github.com/username/repo_name`                   |
| **Authorization callback URL** | `githubstore://callback`                    |

Ardindan **Create application** butonuna tiklayin.

### 2 - Client ID'nizi Kopyalayin
Uygulamayi olusturduktan sonra GitHub su bilgileri gosterecektir:
- **Client ID** ← ihtiyaciniz olan budur
- **Client Secret** ← ❗ Bu proje icin gerekli DEGİLDİR

### 3 - Projenize Ekleyin
Projenizin kok dizinindeki `local.properties` dosyasini acin ve ekleyin:
```properties
GITHUB_CLIENT_ID=YOUR_CLIENT_ID_HERE
```

### 4 - Senkronize Edin ve Calistirin
Projeyi senkronize edin ve uygulamayi calistirin. Artik GitHub ile giris yapabilmeniz gerekir.

### ❗ Onemli Notlar
- `local.properties` **Git'e gonderilmez**, bu nedenle Client ID'niz yerel kalir.
- Bu proje yalnizca **Client ID** gerektirir (Client Secret degil).
- Her gelistirici kendi gelistirme calismasi icin kendi OAuth uygulamasini olusturmalidir.

</details>
