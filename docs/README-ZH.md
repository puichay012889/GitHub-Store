# 🗺️ 项目概述

GitHub Store 是一个面向 GitHub Releases 的跨平台应用商店，旨在简化开源软件的发现与安装。它能自动检测可安装的二进制文件（APK、EXE、DMG、AppImage、DEB、RPM），提供一键安装、更新追踪，并以简洁的应用商店风格界面呈现仓库信息。

基于 Kotlin Multiplatform 和 Compose Multiplatform 构建，支持 Android 和桌面平台。

## 🚀 功能特性

- **智能发现**
    - 首页设有 "Trending"、"Hot Release" 和 "Most Popular" 板块，并提供基于时间的筛选器。
    - 仅展示包含有效可安装文件的仓库。
    - 平台感知的主题评分机制，确保 Android/桌面用户优先看到相关应用。
    - 全新改进的搜索功能，具备更好的相关性排序和性能表现。

- **版本浏览与安装**
    - Release 选择器，可浏览并安装任意版本，而不仅限于最新版。
    - 获取每个仓库的所有 Release。
    - 一键"安装最新版本"操作，另有所有可用 Release 及其安装程序的下拉列表。
    - 手动安装选项，附带自动兼容性检查。

- **丰富的详情页面**
    - 应用名称、版本、"安装最新版本"按钮和分享操作。
    - Star 数、Fork 数、开放的 Issue 数。
    - 渲染的 README 内容（"关于此应用"）。
    - 所选 Release 的 Markdown 格式发行说明。
    - 安装程序列表，附带平台标签和文件大小。
    - 深链接支持 — 通过 URL 直接打开仓库详情。
    - 开发者个人资料页面，可浏览开发者的仓库和活动。

- **应用管理**
    - 直接在 GitHub Store 中打开、卸载和降级已安装的应用。
    - Android：APK 架构匹配（armv7/armv8）、包监控和更新追踪。
    - 桌面端（Windows/macOS/Linux）：将安装程序下载到用户的下载文件夹，并使用默认处理程序打开。

- **收藏的仓库**
    - 保存并浏览您在 GitHub 上收藏的仓库。

- **网络与性能**
    - 动态代理支持，实现可配置的网络路由。
    - 改进的缓存系统，加快加载速度并减少 API 使用量。

- **跨平台体验**
    - Android：原生启动画面、会话过期处理和自适应图标。
    - 桌面端：Linux 上优先支持 AppImage 格式，同时兼容 DEB 和 RPM 格式。
    - 本地化支持 12 种语言：英语、西班牙语、法语、日语、韩语、波兰语、俄语、中文、孟加拉语、印地语、意大利语和土耳其语。

## 🔍 我的应用如何出现在 GitHub Store 中？

GitHub Store 不使用任何私有索引或人工筛选规则。
您的项目只要满足以下条件即可自动出现：

1. **GitHub 上的公开仓库**
    - 可见性必须为 `public`。

2. **最新 Release 中包含可安装文件**
    - 最新 Release 必须包含至少一个具有兼容扩展名的文件：
        - Android: `.apk`
        - Windows: `.exe`, `.msi`
        - macOS: `.dmg`, `.pkg`
        - Linux: `.deb`, `.rpm`, `.AppImage`
    - GitHub Store 会忽略自动生成的源代码归档文件（`Source code (zip)` / `Source code (tar.gz)`）。

3. **可通过搜索 / Topics 被发现**
    - 仓库通过 GitHub 的公共搜索 API 获取。
    - Topics、编程语言和描述有助于排序：
        - Android 应用：如 `android`、`mobile`、`apk` 等 Topics。
        - 桌面应用：如 `desktop`、`windows`、`linux`、`macos`、`compose-desktop`、`electron` 等 Topics。
    - 拥有一定数量的 Star 会增加出现在 Trending/Hot Release/Most Popular 板块的概率。

如果您的仓库满足这些条件，GitHub Store 就可以通过搜索找到它并自动展示，无需手动提交。

## 🧭 GitHub Store 的工作原理（概览）

1. **搜索**
    - 使用 GitHub 的 `/search/repositories` 端点，配合平台定制的查询。
    - 基于 Topics、编程语言和描述进行简单评分。
    - 过滤归档仓库和信号较少的仓库。

2. **Release 和文件检查**
    - 对候选仓库调用 `/repos/{owner}/{repo}/releases/latest`。
    - 检查 `assets` 数组中是否包含特定平台的文件扩展名。
    - 如果未找到合适的文件，该仓库将从结果中排除。
    - 用户还可以通过 Release 选择器浏览所有版本。

3. **详情页面**
    - 仓库信息：名称、所有者、描述、Star 数、Fork 数、Issue 数。
    - Release 浏览器：浏览任意版本及其标签、日期、更新日志和文件。
    - README：从主分支加载并渲染为"关于此应用"。
    - 开发者个人资料链接和分享操作。
    - 可通过深链接直接导航。

4. **安装流程**
    - 当用户点击"安装最新版本"或选择特定 Release 时：
        - 为当前平台选择最合适的文件（Android 上会进行架构匹配）。
        - 流式下载并支持缓存。
        - 委托给操作系统安装程序（Android 上的 APK 安装器，桌面端的默认处理程序）。
        - 在 Android 上，将安装记录到本地数据库，并使用包监控来保持已安装应用列表的同步。
        - 支持对已管理应用的打开、卸载和降级操作。

## ✅ 优势 / 为什么使用 GitHub Store？

- **无需再翻找 GitHub Releases**
  只显示真正为您的平台分发二进制文件的仓库。

- **知晓您的安装记录**
  追踪通过 GitHub Store 安装的应用（Android），并在有新版本发布时高亮提示，让您无需再次搜索 GitHub 即可更新。

- **始终保持最新**
  默认安装最新发布的 Release，同时可通过 Release 选择器浏览并安装任何历史版本。

- **跨平台一致体验**
  Android 和桌面端共享相同的界面和逻辑，安装行为则遵循各平台原生方式。

- **开源且可扩展**
  采用 KMP 编写，网络层、领域逻辑和 UI 层分离清晰 — 易于 Fork、扩展或适配。

## 🔑 GitHub OAuth 配置

**摘要**
1. 创建一个 GitHub OAuth 应用
2. 复制 **Client ID**
3. 将其添加到 `local.properties`

<details>
<summary><strong>查看完整配置指南</strong></summary>

  <br/>

### 1 - 创建 GitHub OAuth 应用
前往：
**GitHub → Settings → Developer settings → OAuth Apps → New OAuth App**

| 字段                          | 值                                       |
| ------------------------------ | ------------------------------------------- |
| **Application name**           | 任意名称（例如 *GitHub Store Dev*） |
| **Homepage URL** | `https://github.com/username/repo_name`                   |
| **Authorization callback URL** | `githubstore://callback`                    |

然后点击 **Create application**。

### 2 - 复制您的 Client ID
创建应用后，GitHub 将显示：
- **Client ID** ← 这是您需要的
- **Client Secret** ← ❗ 本项目不需要

### 3 - 添加到您的项目
打开项目根目录下的 `local.properties` 文件并添加：
```properties
GITHUB_CLIENT_ID=YOUR_CLIENT_ID_HERE
```

### 4 - 同步并运行
同步项目并运行应用。现在您应该可以使用 GitHub 登录了。

### ❗ 重要提示
- `local.properties` **不会提交到 Git**，因此您的 Client ID 仅保留在本地。
- 本项目仅需要 **Client ID**（不需要 Client Secret）。
- 每位开发者应为开发创建自己的 OAuth 应用。

</details>
